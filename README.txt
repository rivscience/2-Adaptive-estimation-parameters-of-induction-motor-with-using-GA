Squirrel-cage induction motor
	
	1 Иногда может вылезать ошибка Error in UniformCrossover (line 4) alpha = unifrnd(-gamma, 1+gamma, size(x1)); Ничего страшного, нажимайте "Запустить" пока она не пропадет. Появилась на этапе разрастания файла "ObjectFunction". Разбираться как это исправить было некогда.
	1.1. 28.10.2022 Понял почему возникает такая проблема - при установки минимальных и максимальных значений искомых переменных возможен подбор таких рандомных значений при которых отклик настраевоемой математической модели будет будет максимально неверным и выйдет за значение ~~10^356. Чтобы избежать такой ситуации необходимо уменьшить диапазон между минимальным и максимальным значением каждой искомой переменной. После этого будет работать стабильно. 	
        
	2 app1 - запуск GA

        3 Obgectfunction - целевая функция по мощности исследуемого двигателя с замкнутой обмоткой ротора

        3 Data_[U, I, w]_a=60_xx.mat - данные снятые с ПЧ. Есть подозрение что немного косячные, но даже на таких данных GA вытаскивает информацию.

        4 Открываем, меняем ссылки для загрузки Data_[U, I, w]_a=60_xx.mat, запускаем app1, насслаждаемся.

        5 Запускаем проверки для отображения графиков из статьи	запускается сам, главное не забыть поменять ссылки.

	6 Для исследования изменения переменных копировал данные с "Command Window", переносил в Excel в папке "Итерация", разбивал по столбцам и переводил в формат ".mat".
